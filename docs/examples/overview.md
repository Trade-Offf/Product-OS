# 完整示例

项目 `examples/` 目录里有一个「会员等级体系」的完整示例，展示从输入到输出的全流程。

建议第一次使用前先翻一遍，看看「输入什么 → 输出什么」。

## 流程四步

| 步骤 | 说明 | 对应文件 |
|------|------|----------|
| 1. 原始需求 | PM 的零散材料，口语化、信息分散 | `examples/sample_input.md` |
| 2. 整理 | AI 结构化整理，标记 5 个待确认问题 | `examples/sample_worksheet.md` |
| 3. 生成 PRD | 待确认清零后生成正式文档 | `examples/sample_prd.md` |
| 4. 精简 | 基于正式 PRD 压缩为 1-2 页速读版 | `examples/sample_prd_light.md` |

## 怎么看

在 Cursor 中直接打开 `examples/` 目录下的文件即可。四个文件按顺序阅读，完整还原了一次需求从「乱」到「清」的过程。

## 要点

- **输入可以很随意**：口语化、格式混乱都行
- **AI 只整理不发明**：每条内容都标注了原文出处
- **待确认是核心机制**：不确定的东西挂起来，不替你做决定
- **正式 PRD 有门槛**：必须把问题答完才放行
