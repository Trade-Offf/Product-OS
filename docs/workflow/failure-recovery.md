# 失败恢复

遇到问题时的处理方式。

## 场景 1：只整理了一半

- 回到 `_WORK_SHEET.md`
- 检查状态块、待确认列表、输入来源
- 补齐后继续，不要跳过工作台直接出 PRD

## 场景 2：要求生成正式 PRD，但仍有待确认

AI 会拒绝正式生成，并提示你：
- 先清零待确认问题
- 或改为生成草稿 `_draft.md`

## 场景 3：需要「现状逻辑」但没有代码文件

- 「现状逻辑」章节会保留
- 标记为 `待扫描`
- AI 不会编造现状逻辑
- 你可以用 `@文件名` 引用代码文件后再生成

## 场景 4：目标输出文件已存在

- 默认不覆盖
- 自动创建新文件名（追加序号 / 版本号 / 日期）

## 场景 5：扩展技能干扰主流程

- 主链路优先：整理 → 状态确认 → 生成/拒绝
- 扩展技能（前端设计、Mermaid 图、原型转换）只能在你明确提出时单独执行
- 不会自动插入主流程
