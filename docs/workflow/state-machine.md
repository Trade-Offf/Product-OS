# 状态机

`_WORK_SHEET.md` 是主流程的唯一状态源。所有状态判断只认这个文件。

## 四种状态

### pending

已完成需求整理，但仍有待确认问题未清零。

- ✗ 禁止生成正式 PRD
- ✓ 允许继续补充问题答案
- ✓ 允许按用户要求生成草稿 `_draft.md`

### ready

待确认问题已清零，工作台已具备正式生成条件。

- ✓ 允许生成正式 PRD
- 正式生成后状态更新为 `completed`

### draft

已生成草稿 PRD，但尚未达到正式交付条件。

- 草稿不等于正式完成
- 待确认仍未清零 → 仍禁止正式生成
- 待确认清零后 → 先更新到 `ready`

### completed

正式 PRD 已生成完成。

- 若再次生成，默认输出新文件，不覆盖原文件

## 状态流转

```
pending → ready（待确认清零）
pending → draft（生成草稿）
draft → ready（待确认清零）
ready → completed（生成正式 PRD）
completed → completed（再次生成，新建文件）
```

## WorkSheet 结构

`_WORK_SHEET.md` 必须包含以下结构：

### 元数据区

- 功能名称
- 当前状态：`pending` | `ready` | `draft` | `completed`
- 当前阶段
- 输入来源
- `Context.md`：有 / 无
- 代码扫描：已扫描 / 待扫描 / 不适用
- 草稿文件路径
- 正式文件路径
- 最后更新时间

### 状态块

- 待确认问题总数
- 已解决问题总数
- 是否允许正式生成：是 / 否
- 是否允许草稿生成：是 / 否
- 阻塞原因

### 内容区

- 需求整理（Part 2）
- 待确认问题（Part 1）
- 代码扫描建议（Part 3）
- 输出记录
