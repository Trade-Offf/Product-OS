# 主链路流程

Product-OS 的主链路围绕 3 个核心对象：

- `01_Input/`：原始输入
- `_WORK_SHEET.md`：主流程唯一状态源
- `03_Output/`：最终输出

## 流程总览

```
原始材料 → [整理] → _WORK_SHEET.md → [回答问题] → [生成PRD] → 03_Output/
```

## Step 1：整理需求

**触发词**：`整理` / `整理一下`

AI 会：
1. 读取 `01_Input/` 中的所有原始材料
2. 如果存在 `01_Input/Context.md`，一并读取
3. 生成或更新 `_WORK_SHEET.md`
4. **不会**在此步骤生成 PRD

整理完成后，`_WORK_SHEET.md` 的状态为：
- `pending`：有待确认问题
- `ready`：待确认问题已清零

## Step 2：确认状态

生成 PRD 前，先看 `_WORK_SHEET.md` 中的状态块：

| 状态 | 含义 | 允许正式生成 | 允许草稿 |
|------|------|:---:|:---:|
| `pending` | 仍有待确认问题 | ✗ | ✓ |
| `ready` | 待确认已清零 | ✓ | ✓ |
| `draft` | 已生成草稿，未达正式条件 | 视待确认 | ✓ |
| `completed` | 正式 PRD 已生成 | 新建文件 | ✓ |

## Step 3：生成 PRD

**触发词**：`生成文档` / `生成PRD`

AI 会根据当前状态决定行为：

### 状态为 `pending`
- 正式 PRD：**拒绝生成**
- 草稿 PRD：允许，输出 `_draft.md`

### 状态为 `ready`
- 允许生成正式 PRD，输出到 `03_Output/`
- 默认不覆盖已有文件

### 状态为 `draft`
- 待确认未清零 → 禁止正式生成
- 待确认已清零 → 先更新为 `ready`，再正式生成

### 状态为 `completed`
- 再次生成 → 创建新文件，不覆盖原文件

## 指令对照表

| 指令/触发词 | 行为 | 结果 |
|---|---|---|
| `整理` / `整理一下` | 执行需求整理 | 只更新 `_WORK_SHEET.md` |
| `生成文档` / `生成PRD` | 尝试生成正式 PRD | 若状态非 `ready`，拒绝 |
| `生成草稿` / `出草稿` | 生成草稿 | 输出 `_draft.md` |
| `完整生成` | 先整理，再等待确认 | 不绕过待确认 |
| `精简一下` / `精简PRD` | 精简正式 PRD | 输出 `PRD_*_light.md` |
| `扫描代码` | 仅在 `@代码文件` 时进行 | 提供现状逻辑依据 |
