# 5 分钟上手

## 你需要准备什么

- 安装好 [Cursor](https://cursor.sh/) 编辑器
- 用 Cursor 打开 Product-OS 项目文件夹

## 三步走完一个需求

### Step 1：把需求丢进去

把你的原始需求材料（飞书文档、会议纪要、语音转文字、零散笔记……格式不限）放进 `01_Input/` 目录。

文件名随意，比如 `需求说明.md`、`老板说的.txt` 都行。

> 如果有历史背景资料（之前做过的版本、遗留决策等），额外放一份 `01_Input/Context.md`。

### Step 2：说「整理」

在 Cursor 对话框里输入：

```
整理
```

AI 会读取 `01_Input/` 里的所有内容，生成一份结构化的 `_WORK_SHEET.md`（工作台）。

工作台会列出：
- 需求目标和范围
- 按端/页面/模块拆解的功能清单
- **待确认问题**（AI 发现的缺口，不会帮你编答案）

### Step 3：回答问题 → 生成 PRD

打开 `_WORK_SHEET.md`，看 Part 1 里的待确认问题。在 Cursor 对话框里直接回答，比如：

```
Q1 回答：降级本期不做，后续再讨论
Q2 回答：图标不阻塞，先用占位图
```

AI 会自动更新 `_WORK_SHEET.md` 里的问题状态。全部回答完之后，输入：

```
生成PRD
```

AI 会在 `03_Output/` 目录生成正式 PRD 文档。

> 如果还有问题没回答完，AI 会拒绝生成正式 PRD，但你可以说 `生成草稿` 先预览结构。

## 常用指令速查

| 你说 | AI 做什么 |
|------|-----------|
| `整理` | 读取输入，生成/更新工作台 |
| `生成PRD` | 生成正式 PRD（待确认必须清零） |
| `生成草稿` | 生成草稿版 PRD（允许有待确认） |
| `精简一下` | 基于正式 PRD 生成精简版（1-2 页） |
| `技能列表` | 查看所有可用技能 |
