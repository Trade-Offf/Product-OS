> 你丢需求，AI 帮你整理成 PRD。

Product-OS 把产品经理放在 `01_Input/` 的原始材料，整理成可追踪、可交付、可恢复的需求主链路文档。

主目标不是「自动补全需求」，而是把已有输入整理为稳定的工作台与交付物，避免流程失控。

## 它能做什么

```
01_Input/（你的需求）→ _WORK_SHEET.md（工作台）→ 03_Output/（PRD）
```

1. **整理**：读取你丢进来的原始材料，生成结构化工作台
2. **追问**：自动识别信息缺口，标记待确认问题
3. **生成**：待确认清零后，输出正式 PRD 文档

## 适用场景

- PM 已经有原始需求材料，但格式不统一
- 原始输入来自飞书文档、会议纪要、语音转文字、零散笔记
- 希望把「待确认问题」「当前状态」「输出文件」放在同一主链路中管理
- 需要明确区分：不能正式生成 / 可以先出草稿 / 可以正式生成

## 不适用场景

- 希望 AI 自行设计完整产品方案
- 希望 AI 在输入缺失时自动补齐业务规则
- 希望跳过工作台直接产出正式 PRD

## 核心原则

- **AI 只整理，不发明**：所有内容来自你的输入，缺口标「待确认」
- **待确认清零才能出正式 PRD**：防止输出不靠谱的文档
- **工作台是唯一状态源**：所有进度看 `_WORK_SHEET.md`
