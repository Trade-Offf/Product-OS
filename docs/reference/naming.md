# 命名规范

## 日期格式

`YYYYMMDD`，例如 `20260407`

## 版本格式

`v1.0`、`v1.1`、`v2.0`

## 功能名

使用下划线，不使用空格。例如 `会员等级体系` → `会员等级体系`

## 文件命名

| 类型 | 格式 | 示例 |
|------|------|------|
| 正式 PRD | `PRD_{功能名}_v{版本}_{日期}.md` | `PRD_会员等级体系_v1.0_20260407.md` |
| 草稿 PRD | `PRD_{功能名}_v{版本}_{日期}_draft.md` | `PRD_会员等级体系_v1.0_20260407_draft.md` |
| 精简版 | `PRD_{功能名}_v{版本}_{日期}_light.md` | `PRD_会员等级体系_v1.0_20260407_light.md` |

## 目录结构

```
Product-OS/
├── 01_Input/           原始输入区（把需求材料丢这里）
├── 03_Output/          最终交付物区（PRD 在这里生成）
├── _WORK_SHEET.md      主流程唯一状态源
├── docs/               文档
│   └── templates/      PRD 模板
├── examples/           完整示例
└── scripts/            工具脚本
```
