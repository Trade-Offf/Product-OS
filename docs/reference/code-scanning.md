# 代码扫描

Product-OS 支持基于实际代码生成「现状逻辑」章节。

## 工作方式

### 有 `@代码文件` 时

在 Cursor 对话中用 `@` 引用代码文件，AI 会：

- 扫描引用的代码
- 「现状逻辑」章节基于实际代码生成
- 每条信息标注代码位置：`(代码位置: 文件名:行号)`

### 没有 `@代码文件` 时

- 「现状逻辑」章节保留
- 标记为 `待扫描`
- **AI 不会编造现状逻辑**

## 使用建议

整理完工作台后，`_WORK_SHEET.md` 的 Part 3 会列出代码扫描建议：

| 功能点 | 建议扫描关键词 | 原因 |
|--------|---------------|------|
| 现有功能 X | 关键词 A、B | 确认现有实现方式 |

在执行 `生成PRD` 时，根据建议用 `@` 引用相关代码文件即可。
