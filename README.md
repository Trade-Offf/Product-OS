## 核心理念

### 架构角色


| 角色      | 载体                               | 职责               |
| ------- | -------------------------------- | ---------------- |
| **主笔**  | Cursor AI                        | 读取多源信息，按模版生成 PRD |
| **图书馆** | `01_Input/Context.md`            | Lark 历史业务背景的注入点  |
| **档案馆** | `@代码文件` 引用                       | Git 仓库中的真实代码逻辑   |
| **蓝图**  | `docs/templates/PRD_Template.md` | PRD 黄金模版         |


### 公式

```
优质 PRD = (Lark历史背景 + Git代码逻辑 + 新需求) × 标准化模版
         = Context.md + @代码文件 + 01_Input/需求.md × PRD_Template.md
```

## 快速开始

### 第一步：打开多根工作区（30 秒）

1. 用 Cursor 打开**业务代码项目**
2. 把 **Product-OS 文件夹**拖到 Cursor 左侧文件树的空白处
3. 弹窗选择 **"Add Folder to Workspace"**

完成后左侧显示两个根目录：

```
业务代码/
  └── src/...

Product-OS/
  ├── 01_Input/    ← 放需求文件
  └── 03_Output/   ← PRD 输出位置
```

现在可以用 `@` 引用业务代码了。

### Step 0: 准备背景（可选但推荐）

**目的**：让 AI 了解历史业务背景，避免"失忆"。

**操作**：

1. 在 Lark 中搜索与本次需求相关的历史文档
2. 复制关键内容
3. 粘贴到 `01_Input/Context.md`

**Context.md 内容建议**：

```markdown
# 业务背景

## 历史沿革
- 2024-01：上线网格交易 v1.0，支持基础买卖
- 2024-06：增加止盈功能，触发即市价单

## 历史决策
- 止盈选择市价单是因为当时流动性不足
- 最小金额 20U 是风控要求

## 遗留问题
- 用户反馈市价单滑点大
- 希望支持限价单止盈
```

**格式要求**：无，随意粘贴即可。AI 会自动提取有用信息。

---

### Step 1: 录入需求

**目的**：将新需求文件放入输入目录。

**操作**：

将需求文件放入 `01_Input/` 目录：

```
01_Input/
├── Context.md          ← Step 0 的历史背景
├── 网格止盈优化.md      ← 新需求（Markdown）
├── 需求评审记录.pdf     ← 会议记录（PDF）
└── 产品脑图.md          ← 其他参考资料
```

**支持格式**：

- Markdown (.md)
- PDF (.pdf)
- 纯文本 (.txt)

**格式要求**：无，随意写。可以是：

- 会议记录
- 脑图导出
- 需求草稿
- 截图描述

---

### Step 2: 整理需求

**目的**：将散乱的输入整理成结构化的工作台文件。

**操作**：

在 Cursor 中对 AI 说：

```
整理一下：我想实现一个xx功能，大致逻辑是...
```

**AI 会自动**：

1. 读取 `01_Input/` 中的所有文件（包括 Context.md）
2. 提取功能点、规则、异常流程
3. 标记逻辑缺口为"待确认问题"
4. 生成 `_WORK_SHEET.md`

**产出文件**：`_WORK_SHEET.md`（项目根目录）

**工作台结构**：

```markdown
## Part 0: 业务背景
（从 Context.md 提取的历史信息）

## Part 1: 待确认问题
### Q1 [⏳ 待回答]
**原文**: "止盈改为限价单"
**问题**: 限价单的价格偏移比例是多少？
**[回答]**: ← 在这里填写答案

## Part 2: 详细拆解
（功能点、规则、异常流程的结构化分析）

## Part 3: 代码扫描建议
（建议生成 PRD 前扫描哪些代码文件）
```

**你需要做的**：

1. 阅读 Part 1 中的待确认问题
2. 在 `**[回答]**:` 后面填写答案
3. 将 `[⏳ 待回答]` 改为 `[✅ 已回答]`

---

### Step 3: 生成 PRD

**目的**：生成正式的产品需求文档。

**操作**：

在 Cursor 中对 AI 说：

```
生成文档
```

**如果涉及已有功能的修改**，需要 `@` 引用相关代码文件：

```
生成文档，扫描 @业务代码/src/main/java/GridStrategy.java 的止盈逻辑
```

**AI 会自动**：

1. 读取 `_WORK_SHEET.md` 中的决策和拆解
2. 读取 `01_Input/Context.md` 中的历史背景
3. 扫描 `@` 引用的代码文件，提取现状逻辑
4. 套用 `docs/templates/PRD_Template.md` 黄金模版
5. 生成完整 PRD

**产出文件**：`03_Output/PRD_{功能名}_v{版本}_{日期}.md`

**PRD 文档结构**：

```markdown
# 网格止盈优化 产品需求文档

## 1. 背景与目标
- 业务背景（来自 Context.md）
- 核心目标（来自新需求）

## 2. 现状逻辑 (Code Truth)
- 当前状态机（来自代码扫描）
- 核心参数（来自代码扫描）
- Mermaid 流程图

## 3. 功能详细设计
- 功能模块
- 字段定义
- 交互规则

## 4. 变更对比
| 维度 | 现状 | 改版后 |
| --- | --- | --- |

## 5. 限制规则
## 6. 异常流程
## 7. 遗留事项
```

#### Step 0: 准备背景

在 Lark 搜索"网格交易"，找到历史文档，粘贴到 `01_Input/Context.md`：

```markdown
# 网格交易历史背景

## 历史版本
- v1.0 (2024-01)：基础网格交易，无止盈
- v1.5 (2024-06)：增加止盈，触发即市价单

## 核心参数
- 最小投入金额：20 USDT
- 网格数量：2-100 格

## 用户反馈
- 市价单滑点大，希望支持限价单
```

#### Step 1: 录入需求

创建 `01_Input/止盈优化需求.md`：

```markdown

用户反馈市价单止盈滑点大

## 需求
1. 止盈支持限价单
2. 用户可设置价格偏移比例（0.1% - 5%）
3. 限价单超时未成交，自动转市价单
```

#### Step 2: 整理需求

对 AI 说：

```
整理一下
```

```markdown
### Q1 [⏳ 待回答]
**原文**: "限价单超时未成交，自动转市价单"
**问题**: 超时时间是多少？
**[回答]**: 30秒
```

```markdown
**原文**: "限价单超时未成交，自动转市价单"
**问题**: 超时时间是多少？
```

## 常见问题

### Q: 为什么要用多根工作区？

A: Cursor 只能读取当前工作区内的文件。如果 PRD 工具和业务代码在不同目录，需要用多根工作区把它们放在一起，才能用 `@` 引用代码文件。

### Q: Context.md 必须填写吗？

A: 不是必须的。但如果不填，PRD 中的"业务背景"章节会缺失历史信息，AI 可能会"失忆"。

### Q: 代码文件必须 @ 引用吗？

A: 如果是**全新功能**，不需要引用代码。如果是**修改已有功能**，建议引用相关代码，让 AI 准确描述"现状逻辑"。

### Q: 如何清除工作区？

A: 执行脚本：

```bash
./scripts/clean_workspace.sh
```

会删除：

- `01_Input/` 中的所有文件（保留 .keep）
- `02_Process/` 中的所有文件
- `03_Output/` 中的所有文件
- `_WORK_SHEET.md`

### Q: 可以同时处理多个需求吗？

A: 建议一次处理一个需求。完成后用 `clean_workspace.sh` 清除，再处理下一个。

### Q: PRD 模版可以自定义吗？

A: 可以。编辑 `docs/templates/PRD_Template.md`，修改章节结构和占位符指令。
