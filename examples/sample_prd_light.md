# 会员等级体系 精简版 PRD

> 基于 `sample_prd.md` 精简，详细内容以原始 PRD 为准。

## 1. 背景 & 目标

- 现有 VIP 体系只有单一级别，用户充值后缺乏成长激励，留存和复购效果有限
- 本次搭建四级会员体系（青铜→白银→黄金→钻石），以积分驱动升级，差异化权益提升留存和转化

## 2. 本次改动范围

- **App 端**：「我的」页面新增等级展示和进度条
- **Web 端**：个人中心同步展示等级信息
- **管理后台**：新增用户等级分布看板 + 手动调级能力
- **后端**：新增等级查询接口、积分变动记录接口、历史积分回填脚本
- **不改**：现有支付流程、商品管理、订单系统

## 3. 关键变更点

- 新增四级会员模型（青铜/白银/黄金/钻石），替代原有普通/VIP 二分模式
- 积分自动累计（消费 + 活跃），达到阈值自动升级
- 各等级差异化权益：折扣优惠、专属客服、生日礼包
- App「我的」和 Web 个人中心展示等级 + 积分进度条
- 管理后台支持查看等级分布和手动调级（需填调整原因）
- 上线时对老用户执行一次历史积分回填

## 4. 数字 & 规则速查

| 项目 | 值 / 规则 | 说明 |
| ---- | --------- | ---- |
| 等级数量 | 4 级 | 青铜→白银→黄金→钻石 |
| 白银阈值 | 500 积分 | |
| 黄金阈值 | 2,000 积分 | |
| 钻石阈值 | 5,000 积分 | |
| 消费积分 | 1 元 = 1 分 | |
| 活跃积分 | 1 天 = 5 分 | |
| 升级方式 | 自动触发 | 积分达标立即生效 |
| 降级 | 本期不做 | 后续讨论 |
| 积分有效期 | 本期不做 | 后续迭代 |

## 5. 风险 / 依赖 / 待定项

- 降级规则未定，本期不纳入，但影响等级生命周期完整性
- 历史积分回填的数据来源和时间范围需开发前确认
- 折扣优惠的商品适用范围和叠加规则需设计评审前确认
- 等级图标依赖设计交付，前端先用占位图开发
- 手动调级时积分是否同步修改需开发前确认
