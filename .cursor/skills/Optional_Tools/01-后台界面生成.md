## 📋 技能依赖声明

### 必须依赖
- ✅ `_WORK_SHEET.md` 文件存在（项目根目录）或
- ✅ `03_Output/PRD_*.md` 文件存在

### 推荐依赖
- 建议先生成PRD文档，界面生成会更准确

### 依赖检查失败时的提示
```

注意：

- 不生成C端/App界面
```

## 🛡️ Safety Protocol (防幻觉协议)

- **Strict Fidelity**: 严格遵循PRD/描述，1:1 还原布局，不擅自优化 UX。
- **No Text Invention**: 如果描述未提及具体文案，使用 "Lorem Ipsum" 占位，**禁止**瞎编内容。
- **Layout Only**: 专注于布局和视觉结构，不添加未提及的交互逻辑（除非明确要求）。
- **Admin Focus**: 只生成后台管理系统界面，不做C端/App界面（由设计师负责）。

## 🔄 跨文件协作协议 (The Loop Protocol)

本技能属于 "增量审计" 环节，严禁直接修改 PRD。

1.  **审计逻辑**：
    - 读取 `_WORK_SHEET.md` (查看已知决策) 和 `03_Output/PRD_Product_Spec.md` (查看当前文档)。
    - 根据本技能的专业视角（后台界面生成）进行检查。

2.  **发现缺口 (Gap Detected)**：
    - 如果发现界面描述不清晰、缺少必要的视觉细节（如表格字段、筛选条件），**严禁自行脑补**。
    - 请将新问题**追加**到 `_WORK_SHEET.md` 文件的 **Part 1** 底部：
      ```markdown
      Q*界面*[序号]: [发现的新问题，如：表格需要展示哪些字段？是否需要分页？]
      [回答]:
      ```

3.  **中断与提示**：
    - 追加问题后，**立即停止**当前任务。
    - 在对话框回复用户：
      "⚠️ 在界面生成中发现设计细节缺失，已追加到 `_WORK_SHEET.md` 底部。
      请去文件中填写 `[回答]`，保存后再次发送 **'生成文档'** 指令，即可获得包含此细节的最新 PRD。"

## Description

专注于生成**后台管理系统**的HTML预览界面，风格参考 Ant Design / Element UI 等行业最佳实践。

**适用场景**：

- ✅ 管理后台（运营后台、CMS、数据管理平台）
- ✅ 内部工具（审批系统、配置中心、监控面板）
- ❌ C端界面（由UI/UX设计师负责）
- ❌ App界面（由UI/UX设计师负责）

**设计理念**：

- **清晰优先**：信息层级清晰，功能一目了然
- **效率为王**：减少点击，提升操作效率
- **专业克制**：配色专业，不花哨，符合B端审美

## 技术栈要求

### 必须使用

- **HTML5**: 标准 HTML 结构
- **Tailwind CSS**: 通过 CDN 引入（v3.x）
- **FontAwesome**: 图标库（通过 CDN）

### 严禁使用

- ❌ React / Vue / Angular 等构建工具
- ❌ npm / webpack / vite 等打包工具
- ❌ TypeScript / JSX / TSX
- ❌ 需要编译的预处理器（SCSS / Less）

### 核心原则

✅ **Single HTML File** - 单文件，双击即可在浏览器打开  
✅ **CDN Only** - 所有依赖通过 CDN 引入  
✅ **No Build Step** - 无需任何构建步骤

### 配色方案

```css
/* 主色调 - 蓝色系 */
```

### 间距规范

- **小间距**: 8px (space-2)
- **中间距**: 16px (space-4)

- **大标题**: 20px / font-bold

## 处理流程

### Step 1: 分析需求

从PRD中提取后台管理系统的界面需求：

- 列表页（表格）
- 详情页（表单）
- 搜索/筛选功能
- 操作按钮（新增/编辑/删除/审批等）

### Step 2: 选择布局模板

根据功能类型选择合适的布局：

| 功能类型 | 布局模板                      | 说明             |
| :------- | :---------------------------- | :--------------- |
| 数据列表 | 搜索栏 + 操作栏 + 表格 + 分页 | 最常见的后台页面 |
| 表单录入 | 面包屑 + 表单卡片 + 操作按钮  | 新增/编辑页面    |
| 详情展示 | 面包屑 + 信息卡片 + 操作按钮  | 查看详情页面     |
| 审批流程 | 步骤条 + 表单 + 审批记录      | 审批页面         |
| 统计面板 | 数据卡片 + 图表区域           | Dashboard        |

### Step 3: 实现组件

**后台常用组件**：

| 组件名称 | HTML实现                         | 说明                      |
| :------- | :------------------------------- | :------------------------ |
| 搜索框   | `<input type="text">` + 搜索按钮 | 支持Enter搜索             |
| 筛选器   | `<select>` 下拉框                | 状态/类型筛选             |
| 操作按钮 | `<button>` + 图标                | 新增/导出/批量操作        |
| 数据表格 | `<table>` + 响应式               | 支持排序/固定列           |
| 分页器   | 分页组件                         | 总数+页码+页大小          |
| 操作列   | 文字按钮组                       | 编辑/删除/详情            |
| 标签     | `<span>` + 背景色                | 状态标签（待审批/已通过） |
| 模态框   | 弹窗组件                         | 二次确认/表单弹窗         |

### Step 4: 输出代码

完整的HTML文件，包含：

1. DOCTYPE 声明
2. CDN 引入（Tailwind CSS + FontAwesome）
3. 后台管理系统布局（侧边栏可选）
4. 功能页面内容
5. 基础交互脚本（原生 JS）

## 注意事项

### ✅ 推荐做法

1. 使用专业的后台配色（蓝色主色调）
2. 表格行高适中（py-4），提升可读性
3. 操作按钮使用文字链接，减少视觉干扰
4. 状态用标签（Tag）展示，颜色语义化
5. 搜索/筛选放在顶部，操作按钮紧随其后

### ❌ 避免做法

1. 不要使用花哨的动画效果
2. 不要使用C端的设计风格（大图、卡片堆叠等）
3. 不要过度装饰，保持简洁专业
4. 不要忽略分页功能（后台必备）
5. 不要使用移动端优先的布局（后台以PC为主）
