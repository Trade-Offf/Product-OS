# Core Flow / 03-PRD精简（可选环节）

> 本技能遵循 `.cursorrules` 中的全局约束。该技能是可选补充环节，不影响主流程。

## 使用场景

- 完整 PRD 对小需求或局部改动来说阅读成本太高
- 上下游同事只需要快速了解：是什么、改了哪里、硬规则、风险依赖

---

## 输入要求

- 必须已存在一份完整 PRD（通常位于 `03_Output/PRD_*.md`）
- 以完整 PRD 为唯一输入源，不直接读 `01_Input/`
- 严格遵守"不新增需求、不修改逻辑"，仅做信息压缩与重排

---

## 输出结构

```markdown
# {项目名称} 精简版 PRD

## 1. 背景 & 目标（短版）
- 2–4 条要点，只保留与本次改动直接相关的背景

## 2. 本次改动范围
- 涉及的端和模块（只列有改动的）
- 不在范围内的内容一句话说明

## 3. 关键变更点
- 3–8 条最重要的变更，每条对应一个可开发/测试的点

## 4. 数字 & 规则速查
| 项目 | 值 / 规则 | 说明 |
| ---- | --------- | ---- |

## 5. 风险 / 依赖 / 待定项
- 从遗留事项中挑 3–5 条与当前版本执行强相关的
```

- 输出路径：`03_Output/PRD_*_light.md`
- 控制在 1–2 页以内

---

## 抽取规则

| 章节 | 来源 | 处理方式 |
| ---- | ---- | -------- |
| 背景目标 | PRD 1.1 + 1.2 | 压缩为 2-4 条要点，删除长段历史叙事 |
| 改动范围 | PRD 3.x + 4.x | 只列有变更的端/模块 |
| 关键变更 | PRD 3.x + 4.x + 5.x | 列表形式，每条 1-2 句 |
| 数字规则 | PRD 5.x + 各章节数值 | 只收硬规则，表格展示 |
| 风险依赖 | PRD 7.x | 挑 3-5 条最相关的 |

---

## 执行步骤

1. **确定目标 PRD**：用户指定或默认使用 `03_Output/` 下最新一份
2. **读取完整 PRD 全文**
3. **按章节抽取信息**，按上方规则压缩
4. **生成精简版 PRD**，开头标注原始 PRD 文件名
5. **写入输出文件**：原文件名追加 `_light` 后缀
6. **回报结果**：告知文件路径，强调精简版只做信息压缩，详细内容以原 PRD 为准

---

## 自检清单

- [ ] 只基于一个完整 PRD，未引用 `01_Input/`
- [ ] 未添加原 PRD 中不存在的新功能或规则
- [ ] 背景目标控制在 2–4 条
- [ ] 关键变更点 3–8 条，可执行级别
- [ ] 数字规则只有硬规则，无灌水
- [ ] 风险/依赖不超过 5 条
